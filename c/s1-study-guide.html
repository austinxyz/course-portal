课后实操指南
把第一节课变成你自己的能力

恭喜你上完第一节课！但 听懂 ≠ 会用。真正的能力，来自你亲手在 cowork 里走一遍。 下面五步，请按顺序做，每一步都别跳过「记录心得」——这是整个方法的灵魂。

材料从哪来 先下载，再开始

开始前，先回到 S1 学员入口页（你点进来的那个页面），下载两样东西：

  1. 课程材料包——点「下载课程材料（含作业）」按钮，下载后 解压。里面有 跟我做-提示词.md、starter-project/、两个 JD，还有 homework-guide.md（这份就是作业要求），下面都会用到。
  2. 课程 PPT——点「下载 Session 1 幻灯片（PPTX）」按钮。
💡 下面第 0 步要放进 ppt/ 的 PPT、要放进 课程心得/ 的 跟我做-提示词.md，以及要放进 作业项目/ 的作业要求（就是解压包里的 homework-guide.md，放进去后改名 作业要求.md），都来自这两个下载。
另外：跟着 跟我做-提示词.md 跑课堂 demo 时，会让你新建 另一个 指向 starter-project 的 Cowork Project——那是独立 Project，和这里的「我的AI课程」并存、随时切换、互不影响。

第 0 步 先搭好你的学习工作区

打开 cowork，新建一个 空目录 作为你的课程学习项目（比如 我的AI课程）。 空的就对了——接下来的内容由你和 AI 一起长出来。

建议的目录结构（边做边建，不用一次到位）：

我的AI课程/
├── 课程心得/
│   ├── ppt/                  # 把课程 PPT 放这里
│   ├── 跟我做-提示词.md       # 下载包里的演示提示词清单，每步带「这步在干嘛」（很有用！）
│   ├── 我的心得.md            # 随手记，想到啥写啥
│   ├── 不懂的名词.md          # 听到没懂的术语/概念
│   └── 课程总结.md            # 让 AI 帮你生成
├── 课堂项目/                  # 跟着课程自己做一遍
└── 作业项目/
    └── 作业要求.md            # 下载包里的 homework-guide.md 放这，改名即可
💡 怕 .md？用 .txt 一样行。 上面那些 .md 文件，你嫌麻烦就全建成 .txt（系统自带记事本就能写）， AI 照样读得懂、照样帮你总结。.md 只是排版能好看一点，不影响用——别让格式挡住你动手。
💬 真实示例（直接抄）——不用自己一个个手动建文件夹，把上面那段结构整段丢给 AI：

「帮我建下面的目录结构，文件我会慢慢放」
（把上面「我的AI课程/…」目录结构整段粘进去）

这步在干嘛：让 AI 一次性把整个课程文件夹骨架建好，你只管之后往里放文件。

第 1 步 记心得 + 让 AI 帮你总结

先你写，再让 AI 补。 顺序很重要——AI 是来帮你深化的，不是替你回忆的。

  1. 在 我的心得.md 里，尽量回忆课上听到的内容，记下你的体会。不用写得完善，碎片、半句话都行。
  2. 在 不懂的名词.md 里，记下你没太懂的术语和概念。
  3. 把 PPT 放进 ppt/ 目录；再把下载包里的 跟我做-提示词.md 也放进 课程心得/——它是课堂 demo 用到的 提示词清单，每步还标了「这步在干嘛」，AI 翻它能准确还原每步在做什么。
  4. 然后让 AI 结合你的心得、PPT 和 跟我做-提示词.md 帮你总结。多问几次，每次问得更细，最后要求它 输出一个 课程总结.md 文件，方便你以后回看。
📎 跟我做-提示词.md 为什么重要：PPT 是「讲了什么」，跟我做-提示词.md 是「实际发了哪些提示词」。 你对某一步没看懂、或想不起 demo 细节时，让 AI 翻 跟我做-提示词.md 回答你，比凭空解释准得多。
Prompt 示例（直接抄，再改成你自己的）：
  • 「请结合我的心得（我的心得.md）、PPT（ppt/ 里的文件）和 跟我做-提示词.md，帮我总结这节课，保存到 课程总结.md。」
  • 「我对 demo 里『___ 那一步』没看懂，请结合 跟我做-提示词.md 给我讲清楚那一步在干嘛、为什么这么做。」
  • 「我对 cowork 里怎么配置 Gmail 不太懂，请给我更详细的步骤指导，并补充到总结文件里。」
  • 「对于『如何写好 prompt』我现在还没概念，请帮我系统梳理一下，加到总结里。」
💡 技巧：把「不懂的名词」逐个丢给 AI，让它用大白话 + 例子解释，再让它写回 不懂的名词.md。 一节课下来，你就有了一份属于自己的术语词典。
💬 真实示例（一位同学真实输入的，直接抄、照着改）——核心是你只管 口述，开文件 / 写入 / 排版全交给 AI：
  • 记一条心得（说完 AI 自动写进 我的心得.md）：
    「将这句话计入我的心得：能指挥 cowork 做的，就让 cowork 做，我是指挥者。」
  • 记一个不懂的名词（AI 解释完自动存进 不懂的名词.md）：
    「我不太懂什么叫做 dispatch。」
  • 不会的操作直接问步骤（问完顺手记进心得）：
    「我对 cowork 里怎么配置 Gmail 不太懂，请给我更详细的步骤指导。」
  • 随时追加新想法：
    「继续加入以下内容到我的心得：提示词可以按自己想法写，写不好就和 AI 讨论让它完善，写错了也能让它重做。」
  • 让 AI 帮你排版：
    「每个心得加个标号？显示得更友好一些。」
一句话窍门：你是指挥者——口述内容，建文件、写入、排版都交给 AI。

第 2 步 亲手做课堂项目

只有自己做，才有真体会。 看录像、对着课程资料，自己在 课堂项目/ 里从头做一遍。

  • 大胆自由发挥——AI 是能容错的，做错了不要紧。
  • 遇到 AI 给的结果不太对时，直接告诉它哪里不对，让它纠正。这个「我说、它改」的来回，本身就是这门课最重要的技能。
  • 做的过程中，继续把心得、踩的坑、突然想通的点，记到 课程心得/我的心得.md 里，让 AI 帮你接着总结。

第 3 步 做作业 = 解决你自己的真实痛点

作业不是交差，是 用这节课的能力解决你日常工作里的一个痛点。项目可大可小。

先想痛点——它可能是：

  • 一件 单调重复 的事（每天导出报表、整理同样格式的文档）；或者
  • 一堆 琐碎零散 的事（没有固定流程，但很烦）。

然后分两种情况：

你的情况怎么做
痛点 已经有固定流程试着把这个流程搬进 cowork，做成一个 agent / workflow
痛点 还没成型把「作业要求 + 你的痛点描述」一起给 AI，让它给你建议方案，保存成文件

关键是迭代——拿到 AI 的建议后，别停。和它一起细化，就像我做 agent 时不断完善 prompt 那样：你提一点、它改一版、你再提……几轮下来，方案就成型了。全程继续写你的心得体会。

Prompt 示例：
「这是我的作业要求（作业要求.md），我日常工作的痛点是 ___。请帮我分析这个痛点适不适合用 AI/cowork 解决，给我 2-3 个方案，保存成 方案.md，并告诉我最简单能先跑起来的是哪个。」
💬 真实示例（填好的样子，照着换成你的痛点）：
「这是我的作业要求（作业要求.md，就是 homework-guide.md），我日常工作的痛点是 要花很多时间写周报。请帮我分析这个痛点适不适合用 AI/cowork 解决，给我 2-3 个方案，保存成 方案.md，并告诉我最简单能先跑起来的是哪个。」

第 4 步 让 AI 帮你给我写一封邮件 = 提交作业

最后，结合你这一路的 心得体会 和 课后作业，让 AI 帮你写一封邮件发给我。

📌 关键说明：这封邮件 就是你的作业提交——「作业要求」邮件里要的每一项内容，邮件里 一项都不能少。 让 AI 帮你写，不等于让 AI 帮你省。所以 prompt 里要把作业要求的每一项都点给 AI，让它逐项填满。

作业要求里要交的（邮件正文必须全部覆盖）：

  1. 工作流设计——你设计了哪几个 Agent，各自职责、数据怎么从上一个传到下一个。
  2. 每个 Agent 的提示词——角色 / 任务 / 输出格式三要素齐全，并写明输入格式和 OUTPUTS/ 输出路径。
  3. 实际运行输出——每个 Agent 真实跑出来的结果（不是占位符、不是「[略]」）。
  4. 学习心得——≥200 字，且至少 3 条具体的个人观察（「这工具好用」不算，要写你真发现了什么）。
Prompt 示例（直接抄，把括号里的路径换成你自己的）：
「请结合我的作业项目（作业项目/ 目录）和课程心得（课程心得/ 目录），帮我写一封发给老师的作业提交邮件。作业要求的每一项内容都必须完整出现在邮件正文里，一项都不能省，按下面四块组织：
  1. 工作流设计：列出我设计的每个 Agent、各自职责、Agent 之间怎么传数据；
  2. 每个 Agent 的提示词：原样贴出，确认角色/任务/输出格式三要素齐全，并带上输入格式和 OUTPUTS/ 路径；
  3. 实际运行输出：把每个 Agent 真实跑出的结果贴进来，不要用占位符；
  4. 学习心得：≥200 字，至少 3 条我自己的具体观察。
末尾再补一段我还想请教的困惑。语气真诚、简洁。写完先列一个清单核对上面四项有没有遗漏。」

我想看到的不是完美，而是 你真的动手了、真的在想。困惑和卡点尤其欢迎——那正是下一节课我能帮你的地方。

一句话总结
写心得 → 让 AI 总结 → 自己做 → 解决真痛点 → 反馈给我。
每一步都亲手做，每一步都留下记录。这门课的能力，是「做」出来的，不是「听」出来的。

—— 看不懂任何一步，截图发学习群，老师和同学帮你。没有笨问题。
